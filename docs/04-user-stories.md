# User Stories with Acceptance Criteria

## US-01 — Patient registration and profile management

- **Derived from:** FR-01
- **Story:** As a Patient, I want patient registration and profile management so that support core medical-record operations in one system.

### Acceptance criteria
- AC-01 — Users can search for a patient by HN, national ID, name, or phone number.
- BR-01 — Patient registration and search shall include duplicate patient detection to reduce duplicate records.
- BR-02 — Medication workflows shall include drug interaction checking and allergy checking.

### Supporting context
- **Workflow reference:** WF-01 — Inpatient discharge documentation
- **Integration touchpoint:** INT-01 — Integrate laboratory results into the medical record
- **Integration touchpoint:** INT-02 — Support PACS and image attachments
- **Business outcome:** Support core medical-record operations in one system

### Delivery notes
- Implement the story so that downstream teams can trace it back to FR-01.
- Preserve validation, error handling, and operational visibility for this story.
